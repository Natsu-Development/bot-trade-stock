## Delegation Discipline (mandatory for every sub-agent spawn)

Canonical phase-to-agent mapping and ready-to-paste prompt templates: **[`.context/policies/delegation-playbook.md`](.context/policies/delegation-playbook.md)**. Assembly helper: **[`scripts/delegate.sh`](scripts/delegate.sh)**.

### The rule (MUST follow)

Before EVERY `Task(subagent_type="oh-my-claudecode:<agent>", prompt=…)` call — **including those issued while running OMC skills** (`/ralplan`, `/ralph`, `/autopilot`, `/team`, `/ultrawork`, `/ultraqa`, `/ultrapilot`) — prepend the matching playbook phase preamble to the `prompt=` argument. The skill's own instructions become the task body; the playbook preamble goes above.

### Assembly

```bash
bash scripts/delegate.sh <phase> --task "<task description>"
# stdout = the assembled prompt → paste into Task(...) prompt argument
bash scripts/delegate.sh --list   # see all phases
```

### Why this is required

OMC plugin skills (`/ralplan`, `/ralph`, `/autopilot`, `/team`, etc.) carry their own internal prompts for sub-agents. Those prompts do NOT reference `.context/` — without this injection, sub-agents work from generic agent system prompts only (no `gitnexus_impact`, no `gitnexus_detect_changes`, no Go Core Rules from `golang-mastery`, no path-specific rules from `.context/rules/backend/`). The injection is leader-enforced; no upstream mechanism does it automatically.

The PostToolUse hook (`scripts/go-check.sh`) still fires on every sub-agent edit regardless — that's the only protection that survives the propagation gap without manual injection.

## Code Intelligence Policy

The GitNexus + gopls workflow (pre-edit impact analysis, gopls inspection, post-edit hook coverage, delegation contract) is consolidated at **[`.context/policies/code-intelligence.md`](.context/policies/code-intelligence.md)**. Read it before any code edit. This policy applies to every code change — leader-direct edits AND sub-agent delegations.

### Critical rules (top-of-file extract; full policy in the file above)

- **MUST** run `gitnexus_impact({target: "<symbol>", direction: "upstream"})` before editing any function, method, class, or exported symbol. Report direct callers, affected flows, risk level (LOW/MEDIUM/HIGH/CRITICAL).
- **MUST** run `gitnexus_detect_changes()` before committing — confirm only intended symbols/flows changed.
- **MUST** warn the user and ask for confirmation if impact analysis returns HIGH or CRITICAL risk before proceeding.
- **NEVER** rename symbols with find-and-replace — use `gitnexus_rename` (graph-aware, cross-language) or `gopls.go_rename_symbol` (Go-only semantic).
- **NEVER** ignore HIGH or CRITICAL risk findings, and never edit a function/method/class without first running impact analysis.
